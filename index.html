Configuração de Firewall

Nesta simulação, você irá configurar um firewall para proteger uma rede corporativa contra ameaças externas.

Cenário:

Você é o administrador de rede de uma empresa que precisa configurar um firewall para proteger os servidores internos. A empresa possui:

  • Servidor web (porta 80 e 443)
  • Servidor de email (porta 25, 110, 143, 465, 587, 993, 995)
  • Servidor de arquivos (porta 445)
  • Servidor SSH (porta 22)

Configure as regras do firewall para permitir apenas o tráfego necessário.

Tarefa 1: Configurar Regras de Entrada

Selecione quais portas devem estar abertas para entrada de tráfego externo:

Porta 80 (HTTP)
Porta 443 (HTTPS)
Porta 21 (FTP)
Porta 25 (SMTP)
Porta 23 (Telnet)

Tarefa 2: Configurar Regras de Saída

Selecione quais portas devem estar abertas para saída de tráfego interno:

Porta 53 (DNS)
Porta 80 (HTTP)
Porta 443 (HTTPS)
Porta 135 (RPC)
Porta 445 (SMB)

Caça às Vulnerabilidades

Encontre vulnerabilidades em um sistema antes que os invasores o façam!

Cenário:

Você é um analista de segurança responsável por encontrar vulnerabilidades em um sistema web antes do lançamento. Examine o código abaixo e identifique as vulnerabilidades:

<?php
$id = $_GET['id'];
$sql = "SELECT * FROM users WHERE id = " . $id;
$result = mysql_query($sql);
?>

Identifique as Vulnerabilidades:

Selecione todas as vulnerabilidades presentes no código:

Injeção SQL
XSS (Cross-Site Scripting)
Uso de função descontinuada (mysql_query)
CSRF (Cross-Site Request Forgery)
Falta de validação de entrada

Glossário de Segurança da Informação

Dicionário completo com termos e conceitos de segurança da informação.

Firewall

Sistema de segurança que controla o tráfego de rede entre redes com diferentes níveis de confiança.

Phishing

Técnica de engenharia social que utiliza mensagens falsas para obter informações sensíveis.

Malware

Software malicioso projetado para danificar ou obter acesso não autorizado a sistemas.

Criptografia

Processo de codificar informações para que apenas destinatários autorizados possam acessá-las.

VPN

Rede Virtual Privada que permite conexão segura através de redes públicas.

Ferramentas de Segurança

Lista de ferramentas úteis para profissionais de segurança.

Wireshark

Analisador de protocolos de rede para solução de problemas e análise de tráfego.

Nmap

Ferramenta de varredura de rede para descobrir hosts e serviços em uma rede.

Metasploit

Framework de teste de penetração para desenvolvimento e execução de exploits.

Burp Suite

Plataforma para teste de segurança de aplicações web.

Nessus

Scanner de vulnerabilidades para identificar falhas de segurança em sistemas.

Checklists de Segurança

Checklists para auditorias e implementações de segurança.

Checklist de Segurança de Rede

Verifique firewalls, segmentação de rede, configurações de VPN e monitoramento.

Checklist de Segurança de Aplicações

Valide autenticação, autorização, validação de entrada e tratamento de erros.

Checklist de Resposta a Incidentes

Prepare plano de resposta, equipe designada e procedimentos de comunicação.

Checklist de Segurança Física

Avalie controle de acesso, vigilância e proteção de equipamentos.

Template para relatar conformidade com políticas e regulamentos.

Relatório de Análise de Vulnerabilidades

Formato para documentar vulnerabilidades identificadas e recomendações.
